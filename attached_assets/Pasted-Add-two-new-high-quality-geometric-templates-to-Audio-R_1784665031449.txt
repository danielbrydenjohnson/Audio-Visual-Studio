NEW TEMPLATE 1: GOLDEN GALAXY

Template name:

Golden Galaxy

Description:

"A spiral galaxy of luminous stars moving through golden-ratio space."

Visual concept:

Create a spiral-galaxy-style template inspired by Fibonacci Spiral, but more cosmic, spacious, and dynamic.

It should feel like a galaxy made from stars, dust, and luminous spiral arms.

It should not look like the existing Fibonacci Spiral copied with different colours.

It should have:

- multiple spiral arms
- golden-angle distribution
- star-like points
- brighter star nodes
- faint dust-like particles
- real 3D depth
- slow forward movement through space similar in feeling to Laser Lattice
- a sense that the viewer is travelling through or past the galaxy
- strong paused-state composition

Use a mathematical structure based on:

- golden angle spacing, approximately 137.5 degrees
- expanding radius
- multiple arm offsets
- z-depth distribution
- varied star brightness
- varied star size
- varied star movement speed

The result should be recognisably galaxy-like when audio is paused.

It should work especially well with Kaleidoscope.

Movement:

Add spatial movement similar to Laser Lattice.

Suggested behaviour:

- stars drift slowly through depth
- some stars and dust move toward the camera
- spiral arms subtly flow locally
- stars recycle or wrap when leaving the depth volume
- dust particles move at different speeds
- motion should feel like travelling through space

Do not rotate the entire template root as the main animation.

Movement should come from individual stars, dust points, arm phase offsets, and depth motion.

Low reaction:

- Low-sensitive stars pulse slightly larger and brighter
- some Low-sensitive stars push outward along their local spiral arm
- some dust particles move faster through depth
- avoid scaling the whole galaxy

Mid reaction:

- Mid-sensitive arm sections bend, twist, or phase-shift locally
- different spiral regions should move differently
- add subtle turbulent flow across the arms
- do not rotate the entire galaxy as one object

High reaction:

- High-sensitive stars sparkle independently
- selected bright stars can flicker or flare briefly
- dust points can twinkle in changing subsets
- avoid full-canvas flashing

Shared controls:

- Visual Density controls star and dust count
- Motion Speed controls depth travel and spiral flow speed
- Element Size controls star and dust size
- Depth controls galaxy thickness and travel depth
- Glow controls luminous intensity
- Brightness affects final output through existing global brightness
- Palette controls colour scheme

Palette behaviour:

- Monochrome should look like white stars in deep black space
- Cyan Violet should create a cosmic neon galaxy
- Ember should create a warm gold/orange galaxy

Use efficient Three.js patterns:

- BufferGeometry
- Points and/or LineSegments
- typed arrays
- shader material if appropriate
- no one mesh per star

NEW TEMPLATE 2: WIREFRAME CORE

Template name:

Wireframe Core

Description:

"A simple nested 3D wireframe shape pulsing and unfolding in space."

Visual concept:

Create a clean central wireframe geometric form.

This should be simpler than a cathedral.

It should be based on one strong recognisable geometric structure rather than a complicated architectural scene.

Good options:

- nested wireframe cubes
- nested octahedrons
- diamond-shaped wireframe shells
- concentric rings
- a central geometric core made from repeated simple frames

The result should feel like a luminous 3D object or machine core.

It should be visually strong when paused.

It should not look like a corridor, cathedral, building, or complex environment.

It should not copy Wireframe Bloom exactly.

Visual requirements:

- central readable geometric shape
- nested layers
- real 3D depth
- clean line work
- strong silhouette
- slight independent movement between layers
- simple enough to understand immediately
- strong with Kaleidoscope enabled

Suggested structure:

- A root group containing a controlled number of wireframe shells
- Each shell can be a cube, octahedron, diamond, ring, or hybrid simple form
- Shells should sit at slightly different scales and z-depths
- Each shell should have its own phase, speed, brightness, Low affinity, Mid affinity, High affinity, and random seed
- Some small node points at corners or intersections are allowed if efficient

Do not create hundreds of individual mesh objects.

Use efficient line geometry:

- LineSegments
- EdgesGeometry where appropriate
- BufferGeometry
- shared materials where possible
- controlled object count

Movement:

The core should move subtly and elegantly.

Suggested behaviour:

- nested shells rotate slowly at different rates
- some shells breathe or flex slightly
- layers drift slightly in depth
- corner nodes or edge highlights shimmer
- movement should be clear but not chaotic

Do not rotate the full root group as the only movement.

Each shell should have its own motion so the structure feels layered.

Low reaction:

- Low-sensitive shells pulse slightly in scale or edge brightness
- selected shells push forward or backward a little
- selected corner nodes may grow briefly
- avoid scaling the complete core uniformly

Mid reaction:

- Mid-sensitive shells twist, rotate, or phase-shift independently
- different layers respond at different rates
- add small local offsets or wobble to selected edges
- avoid rotating everything as one rigid object

High reaction:

- selected edges and corner nodes brighten
- a changing subset of line segments flickers or glints
- high-sensitive details sparkle briefly
- avoid full-canvas flashing

Shared controls:

- Visual Density controls shell count and edge/detail density
- Motion Speed controls shell movement and phase speed
- Element Size controls shell scale and line intensity
- Depth controls z separation and spatial thickness
- Glow controls luminous intensity
- Palette controls colours

Palette behaviour:

- Monochrome should look premium and clean
- Cyan Violet should look futuristic
- Ember should look like glowing copper or gold wire
