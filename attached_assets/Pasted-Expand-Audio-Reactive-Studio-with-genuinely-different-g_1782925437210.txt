NEW TEMPLATE 4: CUBE SWARM

- BoxGeometry

- velocity

- base scale
- Low affinity
- Mid affinity
- High affinity
- random seed

- dominant frequency band

Base movement:

- Cubes drift through 3D space
- Cubes rotate independently
- Cubes move at different speeds
- Cubes recycle or wrap when leaving the volume

Low reaction:

- Low-sensitive cubes briefly increase scale

- Some move faster through depth

High reaction:

IMPLEMENTATION

Implement Kaleidoscope using a WebGL post-processing pass.

Do not duplicate the scene into multiple physical copies.

Do not create multiple cameras arranged in a circle.

Do not use CSS transforms over the canvas.

Use this rendering model:

1. Render the active Three.js scene into a WebGLRenderTarget
2. Render a full-screen triangle or quad
3. Use a custom fragment shader to sample the render target texture
4. Convert UV coordinates around the image centre into polar coordinates
5. Divide the angle into mirrored segments
6. Fold alternating segments to create the kaleidoscope reflection
7. Sample the original rendered image using the transformed coordinates

Use a restrained default of 8 mirrored segments.

Add one additional control visible only when Kaleidoscope is enabled:

Kaleidoscope Segments:
- Options: 4, 6, 8, 10, 12
- Default: 8

Do not add more kaleidoscope controls yet.

The effect should resemble a classic video-editor kaleidoscope effect, not merely mirror the screen once.
