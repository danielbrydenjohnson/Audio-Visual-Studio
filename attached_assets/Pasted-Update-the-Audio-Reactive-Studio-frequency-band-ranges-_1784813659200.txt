GOAL

Change the audio analysis frequency ranges to:

Low:
30 Hz to 100 Hz

Mid:
1,000 Hz to 4,000 Hz

High / Hi-hats:
6,000 Hz to 18,000 Hz

These gaps are intentional:

Ignore:
100 Hz to 1,000 Hz

Ignore:
4,000 Hz to 6,000 Hz

The ignored ranges should not feed Low, Mid, or High.

RATIONALE

Low should now focus tightly on:

- kick
- sub
- bass pressure
- true low-end impact

Mid should now focus more on:

- snare presence
- clap presence
- vocal articulation
- synth presence
- musical detail

High should now focus more on:

- hi-hats
- cymbals
- air
- bright transients

The aim is to reduce muddy or constantly active visual triggering.

IMPLEMENTATION

Update the central frequency band definitions.

Do not hardcode FFT bin numbers.

Continue deriving bin indices from:

- AudioContext sample rate
- fftSize
- frequency range in Hz

Make sure both analysis paths use the updated ranges:

- uploaded track mode
- live input mode

Make sure both signal types use the updated ranges:

- level
- hit / spectral flux

If the analysis code clamps band frequency ranges to Nyquist, keep that behaviour.

If it does not, add safe clamping so the 18,000 Hz upper bound never exceeds the analyser's usable Nyquist limit on lower sample-rate devices.

SENSITIVITY CHECK

Because the ranges are now narrower and more selective, check meter behaviour.

Low:
- should still react clearly to kicks and sub-heavy bass
- should not be constantly active from bass warmth above 100 Hz

Mid:
- should respond to snares, claps, vocals and synth presence
- should be less constantly active than before

High:
- should respond to hi-hats, cymbals and bright transients
- may need slight sensitivity adjustment because 6,000-18,000 Hz can be quieter depending on the track

If needed, lightly retune internal level or flux sensitivity constants.
