CHANGE 2: PRESERVE LEVEL

Keep the existing averaged band energy calculation.

Do not replace it with flux.

Calculate band bin ranges using sample rate and fftSize.

Do not hardcode bin indices.

Use a formula equivalent to:

binWidth = sampleRate / fftSize

binIndex = round(freqHz / binWidth)

or:

binIndex = round(freqHz * fftSize / sampleRate)

Average each band’s bins and convert to a 0 to 100 level value.

Apply downstream smoothing if needed so level remains stable and similar to the current visual feel.

CHANGE 3: ADD SPECTRAL FLUX

Add per-band spectral flux.

Each animation frame:

1. Read current FFT magnitude data.
2. Keep previous FFT magnitude data from the previous frame.
3. For each band, compare current bins with previous bins.
4. Sum only positive differences.
5. Ignore negative differences.
6. Divide by number of bins in the band.
7. Copy current data into previous data for next frame.

Pseudocode:

flux = 0

for each bin in band:
  diff = current[i] - previous[i]

  if diff > 0:
    flux += diff

flux = flux / numberOfBinsInBand

Only rising energy counts.

A sound fading away must not register as a hit.

CHANGE 4: NORMALISE FLUX

Normalise raw flux into a 0 to 1 value.

Use central constants or typed defaults for sensitivity.

A simple approach is acceptable:

normalisedFlux = clamp(rawFlux * sensitivity, 0, 1)

Use sensible per-band sensitivity defaults if required.

Do not expose sensitivity controls yet unless already very simple.

CHANGE 5: HIT ENVELOPE

Shape flux with an attack / decay envelope.

Use frame-rate-independent envelope following.

Persist one hit envelope value per band.

Each frame:

x = normalisedFlux
dt = seconds since previous analysis frame

if x > env:
  coefficient = 1 - exp(-dt / attackSeconds)
else:
  coefficient = 1 - exp(-dt / decaySeconds)

env = env + coefficient * (x - env)

Output hit as:

0 to 100 for display and template use

Use measured dt.

Do not assume 60 fps.

CHANGE 8: MAP HIT INTO TEMPLATES

Now map hit values into the existing working templates.

Do this tastefully.

The goal is not to make everything explode constantly.

General rules:

- level should preserve the existing smooth musical movement
- hit should add punch, impact, sparkle, burst, snap or acceleration
- hit effects should be local and element-specific
- do not make the whole scene flash
- do not scale the entire template root
- do not rotate the entire root as the main reaction
- do not use hit to create ugly jitter
- hit should be noticeable at normal music levels
- hit should decay naturally using the envelope

Existing Low/Mid/High Influence sliders should apply to both level and hit for that band.

If Low Influence is 0%, Low level and Low hit should have no visible effect.

If Mid Influence is 0%, Mid level and Mid hit should have no visible effect.

If High Influence is 0%, High level and High hit should have no visible effect.

If influence is 200%, that band’s level and hit effects should be stronger but still controlled.
